## 🔍 技术细节

### P3.2 ImportExportButtons 关键决策

1. **导入流程 Preview 弹窗确认制**（K 2026-06-17 决策）：
   - file-level 错误（版本不支持/格式无效/根对象缺失/indicators 非数组）→ `message.error` 不进入 Preview
   - indicator-level 错误（4 类：`name_invalid` / `name_duplicate` / `field_invalid` / `parse_error`）→ Preview 弹窗按类型分组展示明细
   - 用户确认后 `importCustomIndicators` 写入 + `IMPORT_CUSTOM_INDICATORS` reducer dispatch

2. **重名检测用 prop 而非 storage**：Preview 阶段用 `customIndicators` prop 检测重名（避免双源数据不一致），**已知限制**：V1.0 接受 prop/storage 不一致时的语义差异，V2.0 优化为"以 storage 为权威源"

3. **导出文件名格式**：`custom-indicators-YYYY-MM-DD.json`（带日期便于版本管理）

### P3.3 CustomIndicatorList 关键决策

1. **引用感知删除**：基于 `isIndicatorReferenced(id, userId)` 区分两种 Popconfirm 文案
   - 未引用：title "确认删除该自编指标？"，description "此操作不可撤销（仅标记软删除）"
   - 已引用：title "该指标被方案引用"，description "删除后引用该指标的条件将自动标记为失效（invalid），是否继续？"
   - **符合 K 偏好**：清晰显示覆盖率异常及原因，不掩盖数据问题

2. **`REMOVE_CUSTOM_INDICATOR` reducer 联动**：删除自动扫描 `filterTree.conditions`，对 `sourceId === id` 的条件标记 `invalid: true, invalidReason: '指标已删除'`（K 2026-06-16 决策已实施）

3. **数据流单向**：UI 组件不直接 dispatch，由 `ConditionBuilder` 父组件在 `onDelete` / `onEdit` 回调中 dispatch
