# 量量日报 — 2026-07-22

## 完成事项

### 1. 修复回测分析「自编指标未找到」错误
- **问题**：Web Worker 内无法访问 `window.localStorage`，`backtestEngine.ts` 在 Worker 中调用 `getCustomIndicatorById()` 返回 `undefined`，导致回测执行时报「未找到自编指标：8条件投票系统」。
- **修复**：在 `BacktestCondition` 中新增 `formula` 字段，配置面板提交时将自编指标公式随配置传入 Worker；引擎直接校验并使用 `condition.formula` 执行，不再依赖 localStorage。
- **影响文件**：
  - `quant-trading-frontend/src/features/backtest/backtestTypes.ts`
  - `quant-trading-frontend/src/features/backtest/BacktestConfigPanel.tsx`
  - `quant-trading-frontend/src/features/backtest/backtestEngine.ts`
  - `quant-trading-frontend/src/features/backtest/backtest.worker.ts`

### 2. 废除回测信号确认机制
- **问题**：引擎仍保留已废除的确认逻辑——要求买入信号必须连续两日成立才执行，导致孤立信号被全部取消；卖出侧同样要求 MA5 持续低于 MA20 才执行。
- **修复**：彻底移除买卖双侧的次日信号确认检查，改为 T+1 执行模型（信号日触发，次日按开盘价/收盘价成交，无论信号是否持续）。
- **影响文件**：`quant-trading-frontend/src/features/backtest/backtestEngine.ts`

### 3. 补充回测诊断信息
- **问题**：无交易时用户无法判断是策略未出信号、被涨停阻挡、资金不足，还是信号出现在期末无法执行。
- **修复**：新增诊断计数器，无完整交易时自动输出买入信号数量、涨停顺延/失效次数、资金不足次数、期末无法 T+1 执行次数，并针对期末信号单独给出明确警告。
- **影响文件**：`quant-trading-frontend/src/features/backtest/backtestEngine.ts`

### 4. 测试与自测
- 重写并新增 `backtestEngine.test.ts` 用例，覆盖孤立信号、期末信号无法执行、完整买卖周期、数据清洗、资金不足等场景。
- 新增 `backtestWorker.test.ts`，覆盖 Worker 输入校验逻辑。
- `npx tsc -b` 通过，单元测试 **34/34** 通过。
- Playwright 浏览器自测：000554 +「8条件投票系统」可正常出结果，期末信号场景下诊断信息准确。
