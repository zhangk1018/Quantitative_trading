### 1. 选股视图架构重构：算与显分离

**背景**：原有架构翻页时每次都向后端请求 + 重新执行自编指标，性能差且状态混乱。

**重构内容**：
- 将筛选流程改为三阶段：批量拉取候选 → 加载 OHLCV → Pyodide 计算自编指标
- 全部筛选完成后再交给前端做纯数组切片分页，翻页零后端请求
- 新增三级缓存机制：候选列表缓存、OHLCV 数据缓存、自编指标结果缓存
- 新增进度条状态管理，实时显示三阶段进度
- 新增熔断机制（指数退避），避免单只股票失败导致整体崩溃

**核心文件**：

### 4. Monaco Editor CDN 加载失败修复

- **问题**：控制台报错 `[getThemeColors] TypeError: Cannot destructure property 'exportedColors' of 'undefined'`

### 5. 自编指标回测无交易问题修复

- **问题**：用"8条件投票系统"选股后点击策略回测，回测无任何交易
- **根因**：项目中存在两套完全独立的自编指标存储系统，数据互不互通
  - 选股器用 `customIndicatorStorage.ts`（ID 格式 `ind_xxx`）
  - 回测用 `scriptStore.ts`（ID 格式 `cs_xxx`）
  - 回测引擎用 `getScript()` 去 `scriptStore` 找选股器创建的指标，结果为 null，导致所有条件判断返回 false
- **修复**：`StrategyBacktestView.tsx` 改用 `getCustomIndicatorById()` 从 `customIndicatorStorage.ts` 读取，统一数据源
- **验证**：用"测试收盘上涨"指标回测 1 年，成功产生 58 次交易
- **文件**：
  - [StrategyBacktestView.tsx](file:///Users/zhangk/workspace/Quantitative_trading/quant-trading-frontend/src/features/strategy-backtest/StrategyBacktestView.tsx)
  - [customIndicatorRunner.ts](file:///Users/zhangk/workspace/Quantitative_trading/quant-trading-frontend/src/features/strategy-backtest/utils/customIndicatorRunner.ts) — 清理无用 import
