## 二、开发任务

### 2.1 自选股页面（watchlist/index.tsx）

| 序号 | 功能 | 描述 |
|------|------|------|
| 1 | 自选股列表 | 表格展示：代码、名称、最新价、涨跌幅、涨跌额、PE、PB、市值、分组 |
| 2 | 实时行情 | 页面加载时调用 `fetchWatchlist` 获取自选股列表，同时拉取最新行情 |
| 3 | 删除自选 | 每行右侧"删除"按钮，确认后调用 `removeOne` |
| 4 | 分组管理 | 筛选/切换分组（默认分组、自定义分组），前端 `useMemo` 过滤 |
| 5 | K线图弹窗 | 双击股票行打开 StockAnalysisModal（复用现有组件） |
| 6 | 添加入口 | 顶部"添加自选"按钮 → 弹出搜索框，输入代码/名称搜索后添加 |
| 7 | 空状态 | 无自选股时显示引导提示"暂无自选股，前往选股器添加" |

### 2.2 组件拆分（容器/展示分离）

| 组件 | 职责 |
|------|------|
| `WatchlistTable` | 表格展示：列定义、数据格式化、涨跌颜色 |
| `WatchlistFilter` | 分组筛选按钮组，切换分组 |
| `WatchlistEmpty` | 空状态引导页 |
| `WatchlistSearchModal` | 搜索添加自选股弹窗 |
| `index.tsx`（主页面） | 布局编排，数据获取，状态分发 |

### 2.3 不涉及的内容
- 后端接口无需改动
- 数据库表无需改动（`user_watchlist` 已建表）
- API 层无需改动
- store 无需改动

## 三、编码规范（K 审阅后采纳项）

### 3.1 性能优化
- 表格行数据用 `useMemo` 缓存派生计算（涨跌幅颜色、PE/PB 格式化），依赖原始行情列表
- 一次性拉取全量自选股，分组筛选作为前端 `useMemo` 派生状态，不触发新请求

### 3.2 安全性
- API 层增加 401 全局拦截跳转登录
- 删除操作加乐观更新 + 回滚机制

### 3.3 可维护性
- 组件拆分：容器/展示分离（见 2.2）
- API 层建立 `WATCHLIST_ERROR_MAP`：409→"已在自选股中"、429→"操作太频繁"、500→"服务器异常"
- 路由常量：`router.tsx` 导出 `ROUTES` 对象，所有跳转引用常量

### 3.4 状态管理
- Loading 状态细化：初次加载（骨架屏）、刷新中（顶部 Spin）、单项操作中（行内 loading 图标）

### 3.5 错误处理
- 空值占位符：统一 `--`，颜色逻辑增加 `isNaN`/`isFinite` 判断
- 删除按钮 `stopPropagation` 防止触发双击事件

### 3.6 测试性
- 所有交互元素添加 `data-testid`：`watchlist-row-{code}`、`watchlist-delete-{code}`、`watchlist-search-input`
- API 层加 `mockError` 模式，模拟超时/500/429 异常
- 监听 `visibilitychange`，后台暂停轮询，前台恢复时立即刷新

## 四、实现记录

### 4.1 后端
- **CRUD 接口**：GET `/api/watchlist`（查询自选股列表）、POST `/api/watchlist`（添加自选股）、DELETE `/api/watchlist/{id}`（删除自选股）、PATCH `/api/watchlist/batch`（批量操作）
- **参数增强**：`stock_codes` 参数支持批量查询，前端可通过逗号分隔的股票代码列表一次性获取多只自选股行情

### 4.2 前端
- **自选股页面** (`quant-trading-frontend/src/features/watchlist/index.tsx`)：完整实现，包含分组管理、实时行情展示、删除功能
- **Store 状态管理** (`quant-trading-frontend/src/features/watchlist/store.tsx`)：通过 React Context + useReducer 实现，提供 `addOne`/`removeOne`/`addMany`/`refresh` 方法
- **组件拆分**：`WatchlistTable`（表格展示）、`WatchlistFilter`（分组筛选按钮组）、`WatchlistEmpty`（空状态引导页）、`WatchlistSearchModal`（搜索添加自选股弹窗）

### 4.3 自选组功能
- **系统分组**（全部/沪深/港股/美股）：不可删除，默认存在
- **自建分组**：用户可新增、删除自定义分组
- **添加自选股时**：下拉分组列表仅显示股票所属市场对应的分组，避免跨市场错误分组

### 4.4 持久化方案
- **localStorage**：分组信息、自选股列表通过 localStorage 持久化存储，刷新页面后数据不丢失
- **同步机制**：前端 Store 与 localStorage 双向同步，确保数据一致性
