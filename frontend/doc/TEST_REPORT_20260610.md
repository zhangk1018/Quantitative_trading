## 三、发现的问题

### 3.1 已记录工单

**[DB-SCHEMA-001] init_db.sql 缺少 user_watchlist 表定义**

- **问题描述**: `init_db.sql` 没有包含 `user_watchlist` 表的 CREATE TABLE 语句，导致在新环境部署时自选股 API（`/api/watchlist/`）返回 500 错误。
- **临时修补**: 手动在 postgres 容器中执行 CREATE TABLE 语句恢复接口。
- **根因**: 初始化脚本与 API 定义不同步。
- **优先级**: P2（中）— 影响新部署，但不影响核心选股功能。
- **处理方**: 量量（后端）

### 3.2 无阻塞性问题

本次测试未发现影响核心功能的阻塞性问题。
