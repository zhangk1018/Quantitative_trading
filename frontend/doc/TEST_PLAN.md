# 量化选股系统测试方案

## 一、测试概述

### 1.1 测试目标
确保量化选股系统的前端应用和后端 API 服务能够稳定、正确地运行，满足业务需求。

### 1.2 测试范围
| 模块 | 测试内容 |
|:-----|:---------|
| 前端应用 | 页面渲染、交互功能、数据展示 |
| 后端 API | 接口响应、数据正确性、性能 |
| Nginx 反代 | 路由转发、静态资源服务 |
| 数据库 | 数据存储、查询效率 |

### 1.3 测试环境
- **前端**: http://localhost/ (Nginx + React SPA)
- **后端**: http://localhost:8000/ (FastAPI)
- **数据库**: PostgreSQL 16 (Docker 容器)

---

## 二、测试策略

### 2.1 测试层次
1. **单元测试**：组件/函数级别的独立测试
2. **集成测试**：前后端接口联调测试
3. **系统测试**：端到端的完整业务流程测试
4. **性能测试**：响应时间、并发处理能力

### 2.2 测试方法
- **手动测试**：UI 交互、流程验证
- **自动化测试**：API 接口测试脚本
- **健康检查**：服务可用性监控

## 五、测试输出

```
测试日期: YYYY-MM-DD
测试环境: 本地 Docker
测试人员: 

┌─────────────────┬─────────┬────────┐
│ 测试项          │ 结果    │ 备注   │
├─────────────────┼─────────┼────────┤

│ API/stocks      │ PASS/FAIL│ ...    │
│ ...             │ ...      │ ...    │
└─────────────────┴─────────┴────────┘

缺陷列表:
1. [问题描述] - [严重程度] - [修复状态]
```

### 5.2 问题分类
| 级别 | 描述 |
|:-----|:-----|
| P0 | 阻塞性问题，系统无法正常运行 |
| P1 | 严重问题，核心功能受影响 |
| P2 | 一般问题，次要功能异常 |
| P3 | 轻微问题，界面/体验问题 |

## 六、测试执行步骤

### 6.1 环境准备
```bash
# 启动后端服务
docker compose --env-file .env.production up -d

# 启动前端服务
docker compose -f docker-compose.frontend.yml --env-file .env.production up -d

# 等待服务启动
sleep 10
```

### 6.2 健康检查
```bash
# 检查容器状态
docker compose ps
docker compose -f docker-compose.frontend.yml ps

# 检查健康端点
curl http://localhost/health
curl http://localhost:8000/health
```

### 6.3 测试执行
参考测试用例文档 `TEST_CASES.md` 执行具体测试用例。

### 6.4 问题记录
发现问题时，记录以下信息：
- 测试用例编号
- 问题描述
- 复现步骤
- 预期结果
- 实际结果
- 严重程度

---

## 七、测试关闭

```bash
# 停止服务
docker compose -f docker-compose.frontend.yml down
docker compose down
```
