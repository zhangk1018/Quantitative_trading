### 2.2 【必须阻塞#1解决】pre_close 与除权除息

**问题**：除权日（如10送10）若使用未复权的 `prev_bar.close` 代替 `pre_close`，会产生虚假±50%涨跌幅，触发错误止损/止盈。

**解决方案（双层保障）**：

1. **后端必须修改（最优）**：snapshot_service.py 的 OHLCV 查询：
   - 增加 `AND adjust_type = 'qfq'` 过滤，确保返回前复权数据
   - 增加 `pre_close` 列（需确认 pre_close 也经过复权调整）
   - 查询变更：`SELECT code, ts, open, high, low, close, volume, pre_close FROM stock_quotes WHERE cycle='1d' AND adjust_type='qfq' ...`
   - OHLCV列顺序变为：`[ts, open, high, low, close, volume, pre_close]`（7列）

2. **前端引擎兜底（若pre_close未复权或后端暂无法加列）**：
   - 引擎使用**前复权 close 序列**计算所有收益率和涨跌停判定
   - 若后端未返回 `pre_close`，引擎使用 `prev_bar.close`（前一根K线复权收盘价）作为昨收价
   - 由于前复权 close 序列是连续的（除权日已做复权调整），`bar[i].close / bar[i-1].close - 1` 即为真实涨跌幅，不存在除权跳空问题
   - UI 需注明："使用前复权价格模拟交易"

> ⚠️ **行动**：Phase 1 编码前需量量确认 snapshot_service.py 的两个改动：①增加 `adjust_type='qfq'` 过滤 ②SELECT 增加 `pre_close` 列。

### 2.3 【必须阻塞#2解决】停牌判定修正

**问题**：一字无量涨停（volume=0, open=high=low=close=涨停价）会被误判为停牌。

**修正方案**：

```typescript
function isSuspended(bar: OhlcvBar, preClose: number, limitPct: number): boolean {
  // 条件1：成交量必须为0
  if (bar.volume !== 0) return false;
  // 条件2：价格区间为0（开盘=最高=最低=收盘）
  if (bar.open !== bar.high || bar.high !== bar.low || bar.low !== bar.close) return false;
  // 条件3（关键）：收盘价不能等于涨跌停价
  // 若收盘=涨停价或跌停价，视为"一字涨跌停无成交"，非停牌
  const upLimit = Math.round(preClose * (1 + limitPct) * 100) / 100;
  const downLimit = Math.round(preClose * (1 - limitPct) * 100) / 100;
  if (bar.close >= upLimit - 0.01) return false;  // 一字涨停
  if (bar.close <= downLimit + 0.01) return false; // 一字跌停
  // 既无成交又不是涨跌停 → 真正的停牌
  return true;
}
```

### 2.4 【强烈建议#3】交易日历

**问题**：用个股OHLCV日期并集会导致回测天数不足（小盘股频繁停牌导致日期缺失）。

**解决方案**：
- **优先方案**：请后端提供交易日历API（或在snapshot/all中返回 `trade_dates` 数组），数据来源为 `trade_calendar` 表（`is_open=1` 的日期）
- **Phase 1 兜底**：取沪深300成分股（代码以60/00/30开头、市值>100亿）的 OHLCV 日期并集，这些大蓝筹几乎不停牌，交集可覆盖99.9%交易日
- 引擎中某股票在某日无OHLCV数据 → 标记为不可交易（suspended或未上市），不影响全局交易日历

### 5.1 OHLCV数据列定义

```typescript
// 后端返回7列（需后端补充pre_close）
const OHLCV_TS = 0;
const OHLCV_OPEN = 1;
const OHLCV_HIGH = 2;
const OHLCV_LOW = 3;
const OHLCV_CLOSE = 4;
const OHLCV_VOLUME = 5;
const OHLCV_PRE_CLOSE = 6;  // 新增列（前复权昨收价）
```

**pre_close获取优先级**：
1. 若后端返回了 `pre_close` 列（第7列）→ 使用之（前提：后端确保pre_close为复权口径）
2. 若后端未返回 → 使用 `bars[i-1][OHLCV_CLOSE]`（前一根K线复权收盘价），因前复权序列连续，除权日无虚假涨跌

### 5.2 交易日历构建

```typescript
function buildTradeDates(allOhlcv: Map<string, number[][]>, marketFilter: string[]): string[] {
  // 优先：如果后端返回trade_dates，直接使用
```

### 5.3 停牌判定（修正版）

```typescript
function isSuspended(bar: number[], preClose: number, limitPct: number): boolean {
  if (bar[OHLCV_VOLUME] !== 0) return false;
  if (bar[OHLCV_OPEN] !== bar[OHLCV_HIGH]
      || bar[OHLCV_HIGH] !== bar[OHLCV_LOW]
      || bar[OHLCV_LOW] !== bar[OHLCV_CLOSE]) return false;
  // 关键修正：排除一字涨跌停
  const close = bar[OHLCV_CLOSE];
  const upLimit = Math.round(preClose * (1 + limitPct) * 100) / 100;
  const downLimit = Math.round(preClose * (1 - limitPct) * 100) / 100;
  if (close >= upLimit - 0.01) return false;   // 一字涨停=涨停封板，非停牌
  if (close <= downLimit + 0.01) return false;  // 一字跌停=跌停封板，非停牌
  return true;
}
```

### 5.4 【强烈建议#4】AST NaN硬阻断

```typescript
function evaluateFilter(node: FilterNode, ctx: StockContext, idx: number): boolean {
  switch (node.type) {
    case 'and':
      return node.children.every(c => evaluateFilter(c, ctx, idx));
    case 'or':
      return node.children.some(c => evaluateFilter(c, ctx, idx));
    case 'not':
      return !evaluateFilter(node.child, ctx, idx);
    case 'range': {
      const val = getFieldValue(ctx, node.field, idx);
      // 关键：NaN/undefined直接返回false，不参与布尔运算
      if (val === null || val === undefined || Number.isNaN(val)) return false;
      const meetsMin = node.min === undefined || val >= node.min;
      const meetsMax = node.max === undefined || val <= node.max;
      return meetsMin && meetsMax;
    }
    case 'pattern': {
      const result = checkTechPattern(node.pattern, ctx.cache, idx);
      // 指标为NaN时pattern检查返回false，checkTechPattern内部已处理NaN
      return result;
    }
    case 'kline': {
      return checkKlinePattern(node.pattern, ctx, idx, node.lookbackDays);
    }
    case 'market': {
      return checkMarket(ctx.stock, node.boards, node.watchlistOnly);
    }
  }
}
```

`getFieldValue` 和 `checkTechPattern` 内部对所有指标取值做NaN检查：
```typescript
function getIndicatorVal(cache: IndicatorCache, field: string, idx: number): number | null {
  const val = cache[field]?.[idx];
  if (val === undefined || val === null || Number.isNaN(val)) return null;
  return val;
}
```

### 5.5 【强烈建议#5】单日最大亏损计算基准

**引擎执行顺序**：

```
for each day i:
  a) 更新持仓股票当日价格
  b) 执行买卖指令（含手续费扣除）→ 更新cash和positions
  c) 计算当日总市值（持仓市值 + 现金）
  d) 日收益率 = (今日总市值 / 昨日总市值) - 1  ← 已包含当日买卖产生的手续费
  e) 检查单日最大亏损：
     if dailyLossLimitEnabled && dayReturn <= dailyLossLimitPct:
       forceLiquidate = true  // 次日空仓
       warnings.push(`Day ${tradeDates[i]}: 单日亏损${(dayReturn*100).toFixed(1)}%，触发清仓`)
```

即单日亏损为**扣除当日所有交易成本（手续费+印花税+滑点）后的实际净值变动率**。

### 5.8 顺延规则（细化版）

**买入顺延**：
```
T日调仓选股→T+1尝试买入
  T+1停牌 → 顺延
  T+1涨停封板（开盘价=涨停价） → 顺延
  T+1正常 → 成交
  顺延超maxDefer天 → 放弃买入（abandon）或按收盘价判断（atClose: 若收盘价未涨停则成交）
  顺延时若股票已不在新一期选股池 → 放弃（避免追高）
```

**卖出顺延**：
```
T日触发卖出→T+1尝试卖出
  T+1停牌 → 无限期等待复牌（不受maxDefer限制），复牌首日开盘价卖出
  T+1跌停封板 → 顺延
  T+1正常 → 成交
  跌停顺延超maxDefer天：
    abandon → 跳过该日继续尝试（不强卖跌停板）
    atClose → 若收盘价未跌停（盘中开板）则按收盘价成交，否则继续顺延
```

### 5.10 每日模拟主循环

```
function runStrategyBacktest(input):
  1. 加载数据：全市场OHLCV（qfq，含pre_close）+ 最新snapshot（含is_st, listed_board）+ 基准OHLCV
  2. 构建全局tradeDates日历（优先后端提供，兜底大蓝筹并集）
  3. [阶段1/2 进度0-70%] 对每只股票预计算IndicatorCache（for循环+Float64Array）
  4. 确定回测起止索引：startIdx = warmupDays, endIdx = tradeDates.length - 1
  5. 确定调仓日列表：rebalanceDays = [startIdx + k*interval for k=0,1,...]
  6. 初始化：cashFen = initialCapital*100, positions=[], equity=[], trades=[], warnings=[]
     peakEquityFen = initialCapital*100, forceLiquidate = false, forceStop = false

  for each day i from startIdx to endIdx:
    a) 更新所有持仓股票当日价格标记
    b) 处理停牌持仓：停牌期间不做止损/止盈检查
    c) 执行待处理买卖指令（顺延队列）：
       - 买入：检查涨停/停牌，可成交则买入（扣除手续费+滑点）
       - 卖出：检查跌停/停牌，可成交则卖出（扣除手续费+印花税+滑点）
    d) 计算当前总市值 = 持仓市值 + cashFen
    e) 非停牌持仓的个股风控检查（止损/止盈/超时）→ 加入卖出队列（T+1执行），标记sellReason
    f) 组合级风控检查（【K建议#5】扣费后的日收益率）：
       if i > startIdx:
         dayReturn = (currentTotalFen - prevTotalFen) / prevTotalFen
         if dailyLossLimitEnabled && dayReturn <= dailyLossLimitPct:
           forceLiquidate = true
           warnings.push(...)
         drawdown = (currentTotalFen - peakEquityFen) / peakEquityFen
         if maxDrawdownStopEnabled && drawdown <= maxDrawdownStopPct:
           forceStop = true
           forceLiquidate = true
    g) 调仓日逻辑（i ∈ rebalanceDays && !forceStop）：
       targetPool = AST过滤器(全部股票, i)  → 排除：已退市、停牌、涨停无法买入
       if forceLiquidate: targetPool = []
       sells = positions.filter(p => !targetPool.has(p.code))  → sellReason='rebalance'
       buys = targetPool.filter(code => !positions.has(code))
       按等权/市值加权计算目标权重，生成买卖指令（T+1执行）
       若1/N > singleStockMaxPct：记录cashDrag警告
    h) 更新净值equityCurve[i]，peakEquityFen
    i) 现金货基收益（若idleCashReturn='moneyMarket'）：cashFen *= (1 + idleCashRate/252)

  最后一天：所有持仓平仓，sellReason='end'
  退市股处理：持仓股OHLCV序列终止→以最后日收盘价强平，sellReason='delisted'
  计算绩效指标（基准不可用时Alpha/Beta/IR返回null）
  返回结果
```
