## 一、设计理念

### 核心原则
1. **专业优先，兼顾易用**：融合聚宽/米筐的专业金融终端风格 + 同花顺的直观易用
2. **暗色主题，数据密集**：暗色背景降低视觉疲劳，优化大量数据展示场景
3. **全链路联动**：选股 → 回测 → 查看K线 → 调参 → 再回测，形成闭环
4. **可扩展架构**：预留实盘交易、组合管理、策略研究等模块扩展位

### 借鉴来源
| 平台 | 借鉴点 |
|------|--------|
| **聚宽 JoinQuant** | 布局范式、因子看板、绩效指标 |
| **米筐 RiceQuant** | 参数配置页结构、回测日志、数据导出 |
| **同花顺 Supermind** | 条件选股搭积木、A股习惯、股票池过滤 |

## 四、选股策略保存/加载

**功能定位**：让用户的选股条件可保存、可分享、可二次加载调试。

**关联文档**：[docs/design/CUSTOM_INDICATOR_AND_STRATEGY_DESIGN.md](file:///Users/zhangk/workspace/Quantitative_trading/docs/design/CUSTOM_INDICATOR_AND_STRATEGY_DESIGN.md)（完整设计）

### 4.1 核心场景

| 场景 | 描述 |
|------|------|
| **保存当前策略** | 用户调好筛选条件 → 一键保存为命名策略 |
| **加载我的策略** | 从列表选择 → 一键恢复所有筛选 + 排序 |
| **分享给他人** | 导出 JSON 文件 / 短码 / 剪贴板 |
| **导入他人策略** | 粘贴短码 / 选文件 → 预览 → 一键导入 |

### 4.2 数据结构

```typescript
interface ScreenerFilters {
  boards: string[];              // 上市地：['main_sh', 'chinext', 'star', 'bse', ...]
  industries: string[];          // 行业：['银行', '地产', ...]
  patterns: string[];            // 形态字段：['pattern_morning_star', 'break_high_20', ...]
  sortBy: string;                // 排序字段
  sortOrder: 'asc' | 'desc';
  topN: number;                  // 取前 N 名
}

interface Strategy {
  id: string;                    // UUID
  name: string;                  // 策略名
  description?: string;
  author?: string;               // 作者（导入时记录来源）
  filters: ScreenerFilters;
  createdAt: string;             // ISO 8601
  updatedAt: string;
}
```

**存储位置**：`localStorage['quant_trading:strategies']`（Phase 1，零后端依赖）

**未来升级**：接口 `StrategyStorage` 已抽象，Phase 2 可无缝替换为 HTTP 实现

### 4.5 导入预览 Modal

**触发**：点击 [📥导入] → 粘贴短码 / 选择 .json 文件

```
┌─ 导入策略 ───────────────────────────────────┐
│                                                │
│  检测到外部策略 "早盘异动"                    │
│  作者: 量量 · 2026-06-07                       │
│  ──────────────────────────────────────────  │
│  ✅ 上市地: 创业板                              │
│  ✅ 行业: （无）                                │
│  ✅ 排序: change_pct DESC · Top 50              │
│  ⚠️  pattern_super_v2 字段不存在，已忽略       │
│  ──────────────────────────────────────────  │
│  策略名: [早盘异动 (导入)________________]     │
│                                                │
│              [取消]      [导入并使用]            │
└────────────────────────────────────────────────┘
```

**设计要点**：
- **不拒绝导入**：字段失效不阻断导入，仅警告
- **预览分类**：✅ 有效字段 + ⚠️ 失效字段一目了然
- **可改名**：导入后策略名可改，默认加 "(导入)" 后缀
- **导入即用**：成功后自动加载该策略，立即可看效果

### 4.6 导入/分享能力矩阵

| 渠道 | 入口 | 实现 | Phase |
|------|------|------|-------|
| **JSON 字符串** | [📥导入] 粘贴 | 直接 JSON.parse | 1 |
| **短码（Base64）** | [📥导入] 粘贴 | URL 安全 Base64 解码 | 1 |
| **.json 文件** | [📥导入] 选文件 | FileReader 读取 | 1 |
| **剪贴板** | [📋复制短码] | navigator.clipboard | 1 |
| **浏览器下载** | [📤导出] | Blob + `<a download>` | 1 |
| **分享链接** | [🔗分享链接] | URL `?import=<code>` 自动检测 | 2 |
| **二维码** | [📱生成二维码] | `qrcode` 库生成 PNG | 2 |

**Phase 1 范围**：JSON 字符串 + 短码 + 文件导入 + 剪贴板 + 浏览器下载
**Phase 2 范围**：分享链接 + 二维码（如用户反馈需要）

### 4.7 关键技术决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 存储位置 | localStorage | 零后端、即时可用；接口已抽象可后续升级 |
| 字段失效处理 | 忽略 + 警告 | 1 个字段失效不应废掉整个策略 |
| ID 冲突处理 | 生成新 UUID + 名字加 "(导入)" | 不覆盖用户本地数据 |
| 分享格式 | Base64 短码 + JSON 文件 | 短码紧凑适合短链，文件可读可存档 |
| 抽屉位置 | 右侧滑出 | 不挡主操作区，符合侧边栏习惯 |
