# 选股策略组合回测 — 方案设计 V2

> 创建日期：2026-07-14
> 作者：方舟
> 状态：待 K 审核后进入开发

## 一、核心定位

将**选股器（StockPicker）**中用户配置的筛选条件（市场范围、行情指标区间、技术形态、K线形态、条件构建器AND/OR/NOT组合）直接转化为**组合回测策略**，在历史300天数据上模拟"每日/每周选股→建仓→持仓→调仓"全过程，输出绩效指标和可视化报告。

---

## 二、设置项分类：系统设置 vs 回测页面

### 原则

- **系统设置**：用户偏好级的默认值，改一次长期生效，影响所有回测
- **回测页面**：单次回测的可变参数，每次运行可临时调整（覆盖默认值）

### 2.1 系统设置（新增第三个Tab"策略回测"）

现有"回测默认设置"Tab保持不变（单股回测V1使用），**新增**"策略回测"Tab专门存放组合回测默认值。

```
系统设置 Tabs:
├── [已有] 涨跌颜色方案
├── [已有] 回测默认设置    ← 单股回测V1（BacktestView）使用
├── [新增] 策略回测        ← 组合选股回测（StrategyBacktestView）使用
└── [已有] 自编指标
```

**"策略回测"Tab 包含 5 个 Card 分组：**

#### Card 1：基础参数（通用）

### 2.2 回测页面（StrategyBacktestView）保留的参数

页面本身**不重复系统设置**，只保留：

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶部策略摘要 | 显示当前选股器配置的条件摘要（市场范围+指标过滤+技术形态+K线形态数量） | 只读展示，"修改选股条件"链接跳转到选股器 |

| **高级设置折叠面板** | 折叠起来的快速调整区，覆盖系统默认值 | 包含：初始资金、调仓频率、最大持仓数、止损、止盈（5个最常调的参数） |
| "使用默认设置"按钮 | 一键重置为系统设置中的值 | — |
| 开始回测 / 取消 | 主操作按钮 | — |

## 四、引擎核心逻辑设计

### 4.1 幸存者偏差处理

**问题**：后端 `/api/snapshot/all` 提供的是**当前交易日**的全市场列式快照（`snapshot_data` 含 listed_status），300天历史数据在 `ohlcv_data` 中（二维数组，按股票×日期）。

**解决方案**：

- 回测引擎对每只股票，在每个交易日检查 `listed_status`：
  - `listed_status=1`（正常上市）→ 可买入、可持有
  - `listed_status=0`（停牌）→ 不可买入、持有中则跳过卖出（顺延至复牌）
  - 股票在 `trade_dates` 序列中消失（退市）→ 退市前最后一个交易日以收盘价强制卖出，之后该股票从选股池移除
- **数据验证**：回测引擎初始化时统计股票池数量变化，输出"退市/新上市股票数"作为警告信息

### 4.2 冷启动预热（250天）

- 数据拉取：请求 `最近交易日 - 250天(预热) - 300天(回测)` 的数据，共 **550天**
- 指标计算：对每只股票一次性计算全周期（550天）的 MA/MACD/RSI/BOLL 数组，缓存到 `Map<stockCode, IndicatorCache>`
- 交易起始：从 `startDate` 对应的日期索引开始模拟交易，前250天仅用于指标计算，不产生交易

### 4.3 按股票缓存指标（性能关键）

```typescript
interface IndicatorCache {
  ma5: Float64Array;   // 长度=总天数(550)，索引i对应tradeDates[i]
  ma10: Float64Array;
  ma20: Float64Array;
  ma60: Float64Array;
  macdDif: Float64Array;
  macdDea: Float64Array;
  macdHist: Float64Array;
  rsi6: Float64Array;
  rsi12: Float64Array;
  rsi24: Float64Array;
  bollUpper: Float64Array;
  bollMid: Float64Array;
  bollLower: Float64Array;
  volRatio5: Float64Array;  // 量比
}
```

- 使用 `Float64Array` 比普通 Array 内存占用更低、访问更快
- 每个调仓日选股时，O(1) 索引取值，无重复计算
- 预计全市场5000只股票指标缓存内存 <200MB（550天×~12指标×8字节×5000股 ≈ 264MB，实际沪深主板有效股票约2500只）

### 4.4 涨跌停/停牌顺延规则

**买入顺延流程**：

```
T+0 调仓日选股 → 尝试T+1开盘买入
  T+1 涨停(开盘价=涨停价 且 成交量极低判定封板) → 记录"买不到"，顺延至T+2
  T+1 正常 → 成交
  T+1 停牌(成交量=0 且 open=close=prev_close) → 顺延
  顺延超 maxDeferDays 天 → 放弃该股买入（资金留作现金）
  顺延期间如果信号消失（该股不再满足选股条件）→ 放弃
```

**卖出顺延流程**：

```
T+0 触发卖出（信号消失/止损/止盈/超时）→ 尝试T+1开盘卖出
  T+1 跌停(开盘价=跌停价) → 顺延，等待开板
  T+1 停牌 → 等待复牌（不受 maxDeferDays 限制，复牌首日开盘价卖出）
  顺延超 maxDeferDays 天（仅跌停场景）→ 按当日开盘价（跌停价）挂单，假设未成交部分按收盘价计（或按 deferFailAction 配置）
```

**涨跌停判定**（前复权价格）：

- 涨停 = `close >= prev_close * 1.10 - 0.01`（主板10%，创业板/科创板20%需按板块区分）
- 跌停 = `close <= prev_close * 0.90 + 0.01`
- 板块涨跌停幅度从 `listed_board` 字段判定：
  - 创业板/科创板：20%
  - ST股（`is_st=1`）：5%
  - 北交所：30%
  - 其他：10%

### 4.5 AST 条件过滤器

将选股器条件树直接序列化为 `FilterNode`，引擎递归执行：

```typescript
type FilterNode =
  | { type: 'and'; children: FilterNode[] }
  | { type: 'or'; children: FilterNode[] }
  | { type: 'not'; child: FilterNode }
  | { type: 'range'; field: RangeField; min?: number; max?: number }
  | { type: 'pattern'; pattern: TechPattern; lookback?: number }
  | { type: 'kline'; pattern: KlinePattern; lookbackDays: number }
  | { type: 'market'; boards?: string[]; watchlistOnly?: boolean };
```

执行器伪代码：

```typescript
function evaluateFilter(node: FilterNode, stock: StockPoint, idx: number, cache: IndicatorCache): boolean {
  switch (node.type) {
    case 'and': return node.children.every(c => evaluateFilter(c, stock, idx, cache));
    case 'or':  return node.children.some(c => evaluateFilter(c, stock, idx, cache));
    case 'not': return !evaluateFilter(node.child, stock, idx, cache);
    case 'range': return checkRange(stock, node.field, node.min, node.max, cache, idx);
    case 'pattern': return checkTechPattern(node.pattern, cache, idx);
    case 'kline': return checkKlinePattern(node.kline, stock, idx, node.lookbackDays);
    case 'market': return checkMarket(stock, node.boards, node.watchlistOnly);
  }
}
```

### 4.6 资金与仓位精度

- 所有金额使用 **整数（分）** 存储和计算，避免浮点误差
- 买入股数 = `Math.floor(可用资金 / 价格 / 100) * 100`（100股一手取整）
- 等权分配时：单股目标资金 = `总资产 / max(持仓数, 1)`，但受限于 `单股最大仓位%` 和 `100股整数倍`
- 剩余现金按 `idleCashReturn` 配置计算：若选"模拟货基"，每日现金产生 `cash * idleCashRate / 252` 的收益

### 4.7 每日模拟循环伪代码

```
function runStrategyBacktest(input):
  1. 加载全市场OHLCV + snapshot数据（550天）
  2. 对每只股票预计算技术指标缓存（Float64Array）
  3. 加载基准指数OHLCV，计算基准净值曲线
  4. 初始化：cash=initialCapital, positions=[], equity=[], trades=[], dailyPnL=[]
  5. 确定回测起始索引（跳过250天预热期）
  6. 确定调仓日列表（按rebalanceFreq筛选周五/月末）

  for each day i from startIdx to endIdx:
    date = tradeDates[i]

    a) 检查组合级风控（单日亏损、最大回撤止损）
       → 若触发：全部清仓标记，后续不再开仓

    b) 处理顺延中的买入/卖出指令
       → 检查今日是否能成交，成交则更新positions/cash

    c) 处理个股风控（止损/止盈/超时）
       → 对每个持仓，检查是否触发，加入卖出队列（T+1执行）

    d) 如果是调仓日 且 未触发组合止损:
       → 应用AST过滤器筛选候选股票池
       → 确定目标持仓（新池 ∩ 现有持仓 → 保留；新池→买入；不在新池→卖出）
       → 生成买卖指令（T+1执行）

    e) 更新当日持仓市值（按收盘价计算）
    f) 计算当日净值、回撤、日收益
    g) 现金收益（如idleCashReturn=moneyMarket）

  7. 回测最后一天：全部清仓
  8. 计算绩效指标（Alpha/Beta/夏普/最大回撤/胜率等）
  9. 返回 BacktestResult
```

## 八、关键风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 后端快照只提供当前截面，无历史snapshot数据 | 幸存者偏差，回测虚高 | 引擎层用listed_status过滤，退市股最后日强平，输出警告 |
| 550天×5000股计算量大，浏览器卡顿 | 用户体验差 | Web Worker + Float64Array缓存 + 进度条+取消按钮 |
| 涨跌停判定需要区分板块 | 顺延逻辑错误 | 按listed_board字段区分涨跌停幅度，ST股单独处理 |
| 100股整数倍导致现金拖累 | 实际仓位偏离等权 | 单股最大仓位%兜底，剩余现金货基选项 |
| 浮点精度问题导致金额计算错误 | 回测结果不准 | 所有金额以"分"为单位整数存储计算 |
| 基准指数数据缺失 | Alpha/Beta无法计算 | 引擎初始化校验基准数据完整性，缺失时跳过基准指标计算 |
