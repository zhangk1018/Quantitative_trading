# 选股策略组合回测 — 方案设计 V3

> 创建日期：2026-07-14
> 作者：方舟
> 版本：V3（吸收K的8条审阅建议 + 数据依赖核查结果）
> 状态：待 K 审核后进入开发

## 一、核心定位

将**选股器（StockPicker）**中用户配置的筛选条件（市场范围、行情指标区间、技术形态、K线形态、条件构建器AND/OR/NOT组合）直接转化为**组合回测策略**，在历史数据上模拟"每N个交易日选股→建仓→持仓→调仓"全过程，输出绩效指标和可视化报告。

## 二、数据依赖核查结果（必须先确认）

### 2.1 后端字段核查结论

经核查 [snapshot_service.py](file:///Users/zhangk/workspace/Quantitative_trading/backend/core/service/snapshot_service.py) 和数据库 Schema [02-schema.sql](file:///Users/zhangk/workspace/Quantitative_trading/docker/postgres/init/02-schema.sql#L50-L66)：

| 数据需求 | 数据库字段 | 当前API返回 | 是否满足 | 处理方案 |
|---------|-----------|-----------|---------|---------|
| OHLCV 历史价格 | `stock_quotes`: open/high/low/close/volume | ✅ 已返回6列 | ✅ 满足 | — |
| 昨收价（计算涨跌停） | `stock_quotes.pre_close` | ❌ 未返回 | ⚠️ 需后端补字段 | **需后端在OHLCV查询中增加pre_close列**（第7列），前端引擎用 prev_bar[OHLCV_CLOSE] 做兜底 |
| 成交额（辅助判断封板） | `stock_quotes.amount` | ❌ 未返回 | ⚠️ Phase 1可省略 | Phase 2补充；Phase 1用volume比率近似判断 |
| 板块信息（涨跌停幅度） | `stock_daily_snapshot.listed_board` | ✅ 最新日已返回 | ⚠️ 近似满足 | 板块不变，用最新日listed_board判定历史涨跌停幅度 |
| ST标识（5%涨跌停） | `stock_daily_snapshot.is_st` | ❌ 查询未SELECT | ⚠️ 近似满足 | **需后端snapshot查询增加is_st字段**；历史ST状态无法回溯，用最新日近似（ST变更频率极低） |
| 停牌状态 | 无独立字段 | ❌ 无 | ⚠️ 前端推断 | 引擎通过 `volume === 0 && open === high && high === low && low === close` 推断停牌日 |
| 退市股处理 | 股票自然从OHLCV序列中消失 | ✅ 自然满足 | ✅ 满足 | 股票序列终止=退市，最后一个交易日以收盘价强制平仓 |
| 涨停/跌停标记 | `stock_daily_snapshot.limit_up/down` | ❌ 查询未SELECT | ⚠️ Phase 1可省略 | 引擎通过pre_close和涨跌幅自行计算 |
| 历史PE/PB/市值等基本面 | 仅最新日snapshot有 | ❌ 无历史截面 | ⚠️ 受限 | **基本面指标（PE/PB/市值/换手率）只能做"当前值筛选"近似**，即使用最新日基本面筛选，无法做"历史某日PE<30"的时点筛选。Phase 1注明此限制 |
| 预热数据长度 | HISTORY_DAYS=300硬编码 | ❌ 固定300天 | ⚠️ 需扩展 | **需后端支持 `?days=N` 参数或扩展到550天**；Phase 1若后端未扩展，则预热天数降级为60天（仅支持MA60），回测可用窗口为240天 |
| 基准指数OHLCV | 无独立接口 | ❌ 无 | ❌ 需新增 | **需后端提供指数日线接口**（或前端首次通过股票K线API获取000300.SH缓存到localStorage） |

### 2.2 Phase 1 启动前必须确认的后端改动

以下3项为**引擎可运行的硬依赖**，编码前需量量协助确认或补全：

1. **OHLCV增加pre_close列**：snapshot_service.py 的 OHLCV 查询增加 `pre_close`，返回格式从6列变为7列
2. **Snapshot查询增加is_st字段**：`SELECT` 中增加 `is_st` 列，引擎用于ST股5%涨跌停判定
3. **基准指数数据**：确认 `/api/kline` 接口是否支持指数代码（如000300.SH），若支持则前端直接获取基准数据

> ⚠️ 如果以上3项暂无法提供，Phase 1引擎可降级运行：
> - pre_close → 用前一根K线的close代替（精度略降，但除除权日外一致）
> - is_st → 默认所有股票10%涨跌停（ST股误判概率低，因ST股代码通常带ST标识，但为稳妥可暂时忽略）
> - 基准 → Alpha/Beta指标暂不计算，显示"基准数据待接入"

## 三、设置项分类：系统设置 vs 回测页面

### 原则

- **系统设置**：用户偏好级的默认值，改一次长期生效，影响所有回测
- **回测页面**：单次回测的可变参数，每次运行可临时调整（覆盖默认值）

### 3.1 系统设置（新增Tab"策略回测"）

现有"回测默认设置"Tab保持不变（单股回测V1使用），**新增**"策略回测"Tab专门存放组合回测默认值。

```
系统设置 Tabs:
├── [已有] 涨跌颜色方案
├── [已有] 回测默认设置    ← 单股回测V1（BacktestView）使用
├── [新增] 策略回测        ← 组合选股回测（StrategyBacktestView）使用
└── [已有] 自编指标
```

**"策略回测"Tab 包含 5 个 Card 分组：**

#### Card 1：基础参数（通用）

| 设置项 | 控件 | 默认值 | 取值范围 | 说明 |
|--------|------|--------|---------|------|
| 初始资金 | InputNumber + "万"后缀 | 100 | 10~10000万 | 回测起始资金 |
| 基准指数 | Select | 沪深300(000300.SH) | 沪深300/中证500/上证50/创业板指 | Alpha/Beta 计算基准 |
| 基准使用全收益指数 | Switch | 关闭 | 开/关 | 开启后使用H00300.CSI（含分红），需后端支持 |
| 无风险利率 | InputNumber + "%"后缀 | 3.0 | 0~10% | 夏普比率计算用 |
| 预热天数 | InputNumber + "天" | 60 | 60~250 | 指标冷启动数据量（MA60需60天，MA250需250天；受后端数据量限制，Phase 1默认60） |

#### Card 2：调仓与仓位

| 设置项 | 控件 | 默认值 | 取值范围 | 说明 |
|--------|------|--------|---------|------|
| 调仓频率 | Radio.Button | 每周 | 每日/每周/每月 | 底层统一为"每N个交易日"：日=1，周=5，月=21 |
| 最大持仓数 | InputNumber | 10 | 3~50只 | 同时持有的最大股票数 |
| 仓位分配 | Radio.Button | 等权 | 等权/市值加权 | 等权=等额分配；市值加权=按市值比例分配 |
| 单股最大仓位 | InputNumber + "%" | 100（关闭） | 5~100% | **默认100%=关闭**。启用后单股仓位=min(1/N, 单股最大仓位%)，剩余现金闲置 |
| 剩余现金处理 | Radio.Button | 闲置 | 闲置/模拟货基(2%年化) | 现金拖累处理方式 |

> ⚠️ **【K建议#1落实】** 等权与单股最大仓位冲突公式：`目标权重 = min(1 / maxPositions, singleStockMaxPct)`。当 `1/N > singleStockMaxPct` 时，剩余现金计入闲置，调仓日输出警告"仓位受限，现金拖累 X%"。

#### Card 3：交易成本

| 设置项 | 控件 | 默认值 | 取值范围 | 说明 |
|--------|------|--------|---------|------|
| 手续费率 | InputNumber + "万分之"后缀 | 2.5 | 0~30 | 双边收取 |
| 滑点 | InputNumber + "万分之"后缀 | 1.0 | 0~30 | 成交价偏移 |
| 印花税 | InputNumber + "‰"后缀 | 1.0 | 0~5 | 仅卖出时收取（千分之一） |
| 最小佣金 | InputNumber + "元" | 5 | 0~50 | 单笔最低佣金 |

#### Card 4：风险控制

| 设置项 | 控件 | 默认值 | 取值范围 | 说明 |
|--------|------|--------|---------|------|
| 止损比例 | InputNumber + "%" | -8 | -20~-2% | 个股亏损触发止损 |
| 止盈比例 | InputNumber + "%" | 25 | 5~100% | 个股盈利触发止盈 |
| 最大持仓天数 | InputNumber + "天" | 20 | 5~120天 | 超时强制平仓 |
| 最大顺延天数 | InputNumber + "天" | 3 | 1~10天 | 涨跌停/停牌后最多等待天数 |
| 顺延失败处理 | Select | 放弃交易 | 放弃交易/按收盘价 | **【K建议#5落实】** partial改为"按收盘价"：超时后若收盘价未涨跌停则按收盘价成交，否则放弃 |
| ─ 组合级风控（默认关闭）─ | Divider + 警示文字 | — | — | **【K建议#2落实】** 以下两项默认关闭，开启时UI弹确认框 |
| 单日最大亏损 | InputNumber + "%" + Switch | **关闭** | -20~-1% 或关闭 | 触发→次日空仓；默认关闭（值为-100%等效关闭） |
| 最大回撤止损 | InputNumber + "%" + Switch | **关闭** | -50~-5% 或关闭 | 触发→全部清仓停止；默认关闭（值为-100%等效关闭） |

> ⚠️ **【K建议#2落实】** 组合级风控（单日亏损/最大回撤）默认关闭，原因：
> - 对历史净值后验截断会天然削峰填谷，高估夏普和卡玛比率（过拟合）
> - 用户首次开启时弹出Modal二次确认："⚠️ 启用组合级风控可能引入过拟合，回测业绩可能偏乐观。实盘中触发即清仓，一旦误判会永久踏空。是否继续？"
> - 绩效报告顶部若检测到启用，显示黄色警告条："已启用组合级风控（单日亏损/最大回撤止损），回测结果包含后验截断效应"

#### Card 5：指标参数（复用）

复用现有"指标参数"Card 中的 MACD/RSI/MA/BOLL/KDJ 设置（共享同一个 `indicatorParams` storage key），无需重复配置。

### 3.2 回测页面（StrategyBacktestView）保留的参数

页面本身**不重复系统设置**，只保留最常调整的参数：

| 区域 | 内容 | 说明 |
|------|------|------|
| 顶部策略摘要 | 显示当前选股器配置的条件摘要（市场范围+指标过滤+技术形态+K线形态数量） | 只读展示，"修改选股条件"链接跳转到选股器 |
| 回测区间 | 开始日期 + 结束日期（DatePicker.RangePicker） | 默认：结束日=最近交易日，开始日=结束日-240天（受300天数据-60天预热限制） |
| **高级设置折叠面板** | 折叠起来的快速调整区，覆盖系统默认值 | **【K建议#4落实】** 包含：初始资金、调仓频率、最大持仓数、止损、止盈、**手续费率、滑点**（7个最常调参数） |
| "使用默认设置"按钮 | 一键重置为系统设置中的值 | — |
| 开始回测 / 取消 | 主操作按钮 | — |

### 3.3 设置项 Storage 定义

新增 `strategyBacktestDefaults` storage key，不影响现有 `backtest_defaults`：

```typescript
// strategyBacktestSettingsStorage.ts（新增文件）
interface StrategyBacktestDefaults {
  // Card 1: 基础
  initialCapital: number;           // 单位：分（存储用分避免浮点，显示用万元）
  benchmarkCode: string;            // '000300.SH'
  benchmarkTotalReturn: boolean;    // 是否使用全收益指数
  riskFreeRate: number;             // 0.03 = 3%
  warmupDays: number;               // 60（受后端数据量限制）
  // Card 2: 调仓与仓位
  rebalanceInterval: number;        // 底层交易日数：1=每日, 5=每周, 21=每月
  maxPositions: number;             // 10
  positionAlloc: 'equal' | 'marketCap';
  singleStockMaxPct: number;        // 1.0 = 100%（即关闭单股上限）
  idleCashReturn: 'none' | 'moneyMarket';
  idleCashRate: number;             // 0.02 = 2%年化
  // Card 3: 交易成本
  feeRate: number;                  // 0.00025 = 万2.5
  slippage: number;                 // 0.0001 = 万1
  stampDuty: number;                // 0.001 = 千1
  minCommission: number;            // 500（分，即5元）
  // Card 4: 风控
  stopLossPct: number;              // -0.08 = -8%
  takeProfitPct: number;            // 0.25 = 25%
  maxHoldDays: number;              // 20
  maxDeferDays: number;             // 3
  deferFailAction: 'abandon' | 'atClose';  // 改为atClose语义
  dailyLossLimitEnabled: boolean;   // 默认false
  dailyLossLimitPct: number;        // -0.05（仅在enabled=true时生效）
  maxDrawdownStopEnabled: boolean;  // 默认false
  maxDrawdownStopPct: number;       // -0.15（仅在enabled=true时生效）
}
```

## 五、引擎核心逻辑设计

### 5.1 【K建议#3落实】调仓日选取：每N个交易日

**不使用"周五选股→周一执行"的自然周逻辑**，统一用交易日计数：

```typescript
function getRebalanceDays(tradeDates: Date[], interval: number, startIdx: number): number[] {
  // interval: 1=每日, 5=每周, 21=每月
  // 从startIdx开始，每interval个交易日选一个调仓日
  const days: number[] = [];
  for (let i = startIdx; i < tradeDates.length; i += interval) {
    days.push(i);
  }
  return days;
}
```

- 调仓**选股**在T日收盘后进行（使用T日及之前数据），**执行**在T+1日开盘价
- 若T+1日停牌或涨跌停无法成交→进入顺延队列
- 选股逻辑使用T日指标，不存在未来函数

### 5.2 幸存者偏差处理

**现实约束**：后端无历史截面snapshot，基本面字段（PE/PB/市值等）只有最新日数据。

**Phase 1方案**：

- **价格/成交量/技术指标**：使用OHLCV历史数据，天然包含退市股到退市日的价格，退市判断为"股票序列终止"
- **退市处理**：股票OHLCV序列终止→判定为退市，最后一个交易日以收盘价强制平仓，卖出原因标记为`delisted`
- **基本面筛选**（PE/PB/市值/换手率）：使用最新日snapshot值近似，回测报告顶部警告"基本面使用最新值"
- **板块筛选**：listed_board用最新日近似（板块变更极罕见）
- **ST标识**：is_st用最新日近似（引擎输出警告："ST标识使用最新日数据，历史ST状态无法精确回溯"）
- **停牌推断**：`volume === 0 && open === high && high === low && low === close` 判定为停牌日
- **新上市股**：OHLCV序列起始日晚于回测起始日→判定为新上市，上市日之前不可买入

### 5.5 涨跌停/停牌顺延规则（细化）

**涨跌停判定**：

```typescript
function getLimitPct(listedBoard: string, isSt: boolean): number {
  if (isSt) return 0.05;
  switch (listedBoard) {
    case 'gem': return 0.20;   // 创业板
    case 'star': return 0.20;  // 科创板（需确认board枚举值）
    case 'beijing': return 0.30; // 北交所
    default: return 0.10;      // 主板
  }
}

function isLimitUp(open: number, preClose: number, limitPct: number): boolean {
  return open >= round2(preClose * (1 + limitPct)) - 0.01;
}

function isLimitDown(open: number, preClose: number, limitPct: number): boolean {
  return open <= round2(preClose * (1 - limitPct)) + 0.01;
}

function isSuspended(bar: OhlcvBar): boolean {
  return bar.volume === 0 && bar.open === bar.high && bar.high === bar.low && bar.low === bar.close;
}
```

**买入顺延流程**：

```
T日（调仓日）选股 → T+1日尝试开盘买入
  T+1停牌 → 顺延
  T+1涨停封板 → 顺延
  T+1正常 → 成交
  顺延超 maxDeferDays 天 → 放弃买入，资金留作现金
  顺延时若股票已不满足选股条件 → 放弃（避免追高）
```

**卖出顺延流程**：

```
T日触发卖出（信号消失/止损/止盈/超时）→ T+1日尝试开盘卖出
  T+1停牌 → 等待复牌（不受maxDeferDays限制），复牌首日开盘价卖出
  T+1跌停封板 → 顺延
  T+1正常 → 成交
  跌停顺延超 maxDeferDays:
    deferFailAction='abandon' → 放弃当日挂单，次日继续尝试（直到成交或不再跌停）
    deferFailAction='atClose' → 若当日收盘价未跌停（盘中开板）→ 按收盘价成交；若仍跌停→继续顺延
```

> ⚠️ **【K建议#5落实】** 无Level2数据，"部分成交"无依据，简化为：
> - `abandon`：顺延超期后跳过该日，次日重新尝试
> - `atClose`：顺延超期后，若收盘价未触及涨跌停则按收盘价成交（假设盘中开板后成交），否则继续顺延
> - 引擎注释和UI说明中明确此假设

### 5.10 每日模拟循环

```
function runStrategyBacktest(input):
  1. 加载全市场OHLCV数据（300天，含pre_close列）
  2. 加载最新日snapshot（含listed_board, is_st, PE/PB/市值/换手率等）
  3. 加载基准指数OHLCV（若可用）
  4. 构建tradeDates全局交易日列表（取所有股票交易日的并集，排序去重）
  5. [阶段1/2] 对每只股票预计算IndicatorCache
  6. 确定回测起始索引 = warmupDays
  7. 确定调仓日列表 = getRebalanceDays(interval, startIdx)
  8. 初始化：cashFen=initialCapital*100, positions=[], equity=[], trades=[], warnings=[]

  for each day i from startIdx to endIdx:
    a) 获取当日所有股票的bar数据
    b) 更新持仓中股票的当前价格
    c) 处理停牌持仓：停牌中不触发止损/止盈（无法卖出）
    d) 组合级风控检查：
       - dailyLossLimitEnabled && 当日亏损超限 → 标记forceLiquidate=true，次日空仓
       - maxDrawdownStopEnabled && 回撤超限 → 标记forceStop=true，清仓并终止
    e) 处理顺延买入/卖出队列：
       - 买入队列：检查今日能否成交（不停牌+不涨停）
       - 卖出队列：检查今日能否成交（不停牌+不跌停）；停牌卖出不设超时
    f) 个股风控检查（非停牌持仓）：
       - 止损/止盈/超时 → 加入卖出队列（T+1执行），记录sellReason
    g) 调仓日逻辑（i ∈ rebalanceDays && !forceStop）：
       - 应用AST过滤器，筛选候选股票池
       - 排除：已退市、停牌、涨停无法买入的
       - 若forceLiquidate：目标池为空（全部卖出）
       - 确定目标持仓：目标池∩现有持仓→保留；不在目标池→卖出(rebalance)；新入池→买入
       - 等权分配：calcTargetWeight，生成买入/卖出指令（T+1执行）
    h) 计算当日持仓市值、总资产、净值、回撤、日收益率
    i) 现金收益（idleCashReturn='moneyMarket'时）：cashFen *= (1 + idleCashRate/252)
    j) 记录equityCurve点

  9. 回测最后一天：全部平仓，sellReason='end'
  10. 若forceStop触发，记录warnings
  11. 计算绩效指标（Alpha/Beta等需基准数据，无基准则跳过）
  12. 返回 StrategyBacktestResult
```

## 九、关键风险与应对

| 风险 | 影响 | 应对方案 |
|------|------|---------|
| 后端无历史截面snapshot | 基本面/PE/PB/市值只能用最新值 | 报告顶部警告"基本面使用最新值，非历史时点"；Phase 3补充历史snapshot |
| 后端HISTORY_DAYS=300硬编码 | 预热天数受限为60天 | Phase 1默认60天预热，MA250在Phase 2后端扩展后启用 |
| 无独立停牌字段 | 停牌判定可能误判（极端一字涨停也可能成交量=0） | 结合涨跌幅判定：若volume=0且close=prev_close(1±10%)，才判定停牌；一字涨停视为涨停非停牌 |
| 无Level2数据 | 部分成交无法模拟 | deferFailAction='atClose' 按收盘价假设，UI明确说明 |
| 历史is_st无法回溯 | ST股5%涨跌停判定有误差 | 用最新日is_st近似，输出警告；ST股代码通常带*ST/ST标识，可二次验证 |
| 5000股×300天计算量大 | 浏览器卡顿 | Web Worker + Float64Array + 两阶段进度条 + 取消按钮 |
| 100股整数倍+单股上限导致现金拖累 | 实际仓位低于等权 | 引擎输出cashDrag警告；货基选项模拟现金收益 |
| 组合级风控过拟合 | 业绩虚高 | 默认关闭；开启时二次确认+报告警告 |
| 浮点精度 | 金额计算错误 | 所有金额用"分"整数存储 |
| 选股器Proxy对象传递 | 序列化失败 | JSON.parse(JSON.stringify(filterTree))深拷贝 |
