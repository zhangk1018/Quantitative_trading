# 回测分析 — 方案 v2.2

> 最终版，融合三份专家审阅意见 + 三位专家反馈修正

## 核心设计原则

1. **消除三类失真**（偷价 / 未来函数 / 不可交易场景）— V1 最高优先级
2. **计算与渲染分离**（Web Worker + 主线程 UI）— 性能底线
3. **原子层复用、场景层独立**（不回染选股组件）— 工程边界
4. **可序列化数据模型**（JSON Schema 承载条件/交易/结果）— 为 V2/V3 预留扩展
5. **一切可复现**（标准化公式 + 基准用例 + 自校验）— 质量底线
6. **配置优先**（指标参数/无风险利率/手续费等全部可配置）
7. **不掩盖数据问题**（三级警告面板，数据异常显式化）

## 一、回测引擎

### 1.1 交易信号时序模型

```
信号确认日（T日）  →  成交日（T+1日）  →  成交价（T+1日开盘价）

K线数据范围：T-N ～ T 日（用于指标计算）
信号判定：基于 T 日及之前所有数据，在 T 日收盘后确认
成交执行：T+1 日开盘价
```

### 1.2 交易状态机

```
                    ┌─────────────────────────┐
                    │        IDLE（空仓）       │
                    └───────────┬─────────────┘
                                │ 买入信号触发
                    ┌───────────▼─────────────┐
                    │      HOLDING（持仓）     │
                    └───────────┬─────────────┘
              ┌─────────────────┼─────────────────┐
              │ 卖出信号触发     │ 回测期末          │ 涨跌停/停牌无法买入
              ▼                 ▼                  │ （顺延≤3天）
         ┌─────────┐     ┌───────────┐            │
         │  IDLE   │     │  CLOSED   │  ◄────────┘
         └─────────┘     └───────────┘
```

### 1.3 交易规则

| 规则 | 说明 |
|------|------|
| 交易单位 | 100 股/手，买入股数 = floor(可用资金 / (成交价 × 100)) × 100 |
| 成交价 | T+1 开盘价（默认） |
| 涨跌停顺延 | 最多顺延 3 个交易日，超时信号失效 |
| 停牌处理 | 跳过信号检测和成交，净值按前一日收盘价估值 |
| 停牌警告 | 超过 30 天触发警告 |
| 信号确认 | 连续 2 根 K 线满足条件 |
| 信号存续 | 确认后至成交前，条件消失则信号失效 |
| 买卖冲突 | 持仓状态下同日同时满足买卖条件 → 卖出优先 |
| 重复信号 | 持仓状态下重复买入信号 → 忽略 |
| 期末清仓 | 最后交易日收盘价强制清仓，不计入胜率 |

### 1.4 买卖配对

| 规则 | 说明 |
|------|------|
| 配对方式 | 按时间顺序 FIFO（第一个买入配第一个卖出） |
| 期末清仓 | 不计入胜率/盈亏比，单独标注 |
| 输出指标 | 区分"已平仓交易次数"和"期末强制清仓" |

### 1.5 反向条件映射

| 买入条件 | 卖出条件 |
|---------|---------|
| MACD 金叉 | MACD 死叉 |
| RSI 超卖（<30） | RSI 超买（>70） |
| 放量突破（vol_ratio_5≥1.5） | 缩量跌破（vol_ratio_5<0.5） |
| 连续上涨（consec_up_days≥3） | 连续下跌（consec_down_days≥3） |
| AND 组合 | 任一条件反向即卖出 |

配置面板自动填充反向条件，用户可手动覆盖。

### 1.6 数据质量规则

| 场景 | 处理规则 |
|------|---------|
| 复权口径 | 统一前复权（adj=forward） |
| K线周期 | V1 仅日线 |
| 预热期 | 用户选定周期 + 60 天（最大指标窗口） |
| 指标窗口 | 前 N 根不足窗口期的不参与信号检测，记录剔除数量 |
| 缺失K线 | 前向填充（OHLC=前一日收盘价，volume=0） |
| 涨跌停 | 主板 ±10%、创业板/科创板 ±20% |

### 1.7 回测引擎纯函数签名

```typescript
interface BacktestInput {
  bars: KlineBar[];
  buyConditions: Condition[];
  config: {
    capital: number;       // 初始资金
    feeRate: number;       // 手续费率
    slippage: number;      // 滑点
    riskFreeRate: number;  // 无风险利率
  };
}

interface BacktestOutput {
  trades: Trade[];
  equityCurve: EquityPoint[];
  summary: BacktestSummary;
  warnings: string[];
}
```

## 二、绩效指标体系

### 2.1 标准化公式

| 指标 | 公式 | 参数 |
|------|------|------|
| 总收益率 | (期末净值 - 初始资金) / 初始资金 × 100% | — |
| 年化收益率 | (1 + 总收益率)^(252/交易天数) - 1 | 252 交易日，复利 |
| 胜率 | 盈利交易次数 / 已平仓交易次数 × 100% | — |
| 盈亏比 | 平均单笔盈利 / abs(平均单笔亏损) | — |
| 最大回撤 | max(1 - 当日净值/此前最高净值) | 日度收盘净值 |
| 最大连续亏损 | 连续亏损交易的最大次数 | — |
| 平均持仓天数 | 已平仓交易持仓天数的均值 | — |
| 夏普比率 | (年化收益率 - 无风险利率) / 年化波动率 | 无风险利率=3%，简单收益率 |
| 交易次数 | 已平仓交易次数 + 期末强制清仓（区分展示） | — |
| 基准收益 | 同期买入持有收益率 | T+1 开盘买入 |

### 2.2 收益口径

V1 采用**价格收益**（基于前复权价格），不含分红送转。UI 标注"价格收益"。

### 2.3 结果自校验

| 校验项 | 规则 |
|--------|------|
| 净值一致性 | 期末净值 = 初始资金 + 所有交易盈亏之和 |
| 交易配对 | 买入次数 = 卖出次数（含期末清仓），时间顺序配对 |
| 指标计算 | 关键指标抽样 3 个时点与独立手算对比 |

## 三、架构设计

### 3.1 计算架构

```
主线程 (UI)                    Web Worker (计算)
┌──────────────┐              ┌──────────────────┐
│ 配置面板      │──postMessage→│ 指标计算          │
│ K线图表       │              │ 信号检测          │
│ 指标卡片      │←─result─────│ 模拟交易          │
│ 交易明细      │←─progress───│ 进度回调          │
└──────────────┘              └──────────────────┘
```

### 3.2 组件复用策略

```
公共原子层（不修改）
├── KLineChart（仅渲染，买卖标记通过外层叠加 canvas 实现）
├── indicators.ts（纯函数）
├── condition-detector.ts（纯函数）
└── 条件原子组件（单个指标选择器/范围输入）

回测场景层（新建）
├── BacktestView
├── BacktestConfigPanel
├── BacktestChart（叠加层）
├── BacktestMetrics
├── BacktestEquityCurve（双Y轴）
├── BacktestTradeLog（虚拟滚动）
└── BacktestWarningBar
```

### 3.3 状态管理

```
IDLE → FETCHING → CALCULATING → FINISHED
  │        │           │            │
  └────────┴───────────┴────────────┘
                  │
               ERROR / CANCEL
```

### 3.4 IndexedDB 存储

| 规则 | 说明 |
|------|------|
| 上限 | 20 条（FIFO） |
| Key | stockCode_timestamp |
| 清理 | 底部"清空本地缓存"按钮 |
| 降级 | 存储失败弹窗提示，不影响结果展示 |

## 五、迭代规划

| 阶段 | 内容 | 验收标准 |
|------|------|---------|
| V1 | 单只股票 + ≤5条件AND + 日线 + 前复权 + T+1开盘 + Web Worker + 完整指标 + 图表 + 本地存储 | 8 基准用例通过，2500根<10s，主线程<50ms |
| V1.5 | OR/NOT组合 + 信号确认可配置 + CSV/JSON导出 | 组合条件可复现 |
| V2 | 仓位管理 + 手续费 + 滑点 + 止盈止损 | 摩擦成本对比 |
| V3 | 多股票批量 + 策略对比 + 服务端持久化 | 批量无内存泄漏 |

## 六、基准校验用例（8个）

| # | 用例 | 预期输出 |
|---|------|---------|
| 1 | 连续30天涨1%，MACD金叉买入 | 持有期收益≈30%，误差<1% |
| 2 | 连续30天跌1%，无买入信号 | 交易次数=0 |
| 3 | 横盘30天，无指标触发 | 交易次数=0 |
| 4 | 金叉买入，10天后死叉卖出 | 交易次数=1，收益=10日涨跌幅 |
| 5 | 金叉当日涨停，次日开盘涨停 | 推迟=1，成交价=第三日开盘 |
| 6 | 金叉日停牌 | 推迟至复牌日 |
| 7 | 回测期内含分红送股 | 前复权价格连续 |
| 8 | MACD金叉+RSI<30同日满足 | 仅买入一次 |

## 八、V1 交付清单

- 回测引擎（Web Worker，纯函数，状态机，A股整手）
- 条件配置（≤5条件AND，原子组件复用，反向条件自动填充）
- 成交规则（T+1开盘，涨跌停顺延≤3天，停牌跳过，信号失效）
- 信号确认（连续2根K线，信号存续校验）
- 数据校验（前复权，预热期60天，窗口剔除，缺失前向填充）
- 图表（K线+买卖标记叠加层，资金曲线双Y轴，X轴联动）
- 指标卡片（10项，含基准收益、盈亏比、平均持仓天数）
- 交易明细（虚拟滚动，点击定位K线，配对标注）
- 警告面板（三级：黄/蓝/红）
- 结果管理（IndexedDB≤20条，多结果对比）
- 基准校验（8个用例，误差<1%，3类自校验）
- 配置化（指标参数可配，无风险利率3%，高级设置面板）
