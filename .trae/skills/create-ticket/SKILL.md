# 开协作单（Create Ticket）

当你发现一个问题需要正式记录并通知方舟时，按以下流程操作。

## 触发条件

- 发现 Bug（数据异常、接口报错、逻辑缺陷）
- 发现系统性问题需要方舟确认或配合
- 需要记录一个待处理的任务（即使自己会修）

## 步骤

### 1. 分析问题

确认：
- **影响范围**：frontend / backend / both
- **优先级**：P0(紧急阻塞) / P1(高) / P2(中) / P3(低)
- **定位路径**：文件路径 + 行号
- **复现方式**：最小复现步骤

### 2. 生成工单 ID

格式：`[阶段-关键词-YYYYMMDD]`
- 阶段：如 `4.4`、`5.2`、`6.1`
- 关键词：大写英文，描述问题域（如 `STOCKS`、`META`、`SCHEMA`）
- 日期：当天日期

示例：`[4.4-STOCKS-20260610]`、`[6.1-SIGNAL-20260610]`

### 3. 写入 docs/协作单.md

在文件末尾追加新工单，使用以下模板：

```markdown
---

## [ID] 简短标题

| 字段 | 内容 |
|------|------|
| **状态** | NEW |
| **提单方** | 量量 |
| **日期** | YYYY-MM-DD |
| **影响范围** | frontend / backend / both |
| **优先级** | P0 / P1 / P2 / P3 |

**定位路径**: 文件路径：行号

**问题描述**:
清晰描述现象、触发条件和影响。

**预期修复**:
描述期望的修复方式。

**验收方式**:
描述如何验证修复是否有效。

---

**处理记录**:
- YYYY-MM-DD 量量: 提单（NEW）
```

### 4. 在 .trae/topics.md 追加通知

在文件末尾追加一行：

```
[量量 YYYY-MM-DD HH:mm] 协作单 [ID] 状态变更: NEW（简短描述问题）
```

## 示例

发现 `/api/stocks/` 接口返回 500：

1. 分析：backend 问题，P0 优先级
2. 生成 ID：`[4.4-STOCKS-20260610]`
3. 追加到 协作单.md 末尾
4. 在 topics.md 追加：`[量量 2026-06-10 14:30] 协作单 [4.4-STOCKS-20260610] 状态变更: NEW（/api/stocks/ 返回 500，jsonable_encoder 序列化失败）`

## 重要规则

- **一个工单一个 ID**，不要重复使用
- **状态变更即通知**：每次改状态都要同步写 topics.md
- **closed 的工单不移除**，保留在 协作单.md 作为历史记录
- 如果自己修复，走"处理协作单"流程（ASSIGNED→VERIFY）
