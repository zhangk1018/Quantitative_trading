# Dogfood Testing

This skill enables systematic internal testing and quality assurance for web applications.

## Testing Phases

### 1. Planning
- Define testing scope and objectives
- Identify test scenarios and use cases
- Set up testing environment

### 2. Execution
- Perform functional testing
- Conduct usability testing
- Test edge cases and boundary conditions

### 3. Reporting
- Document defects and issues
- Capture screenshots and videos
- Generate structured reports

## Testing Types

### Functional Testing
- Verify features work as expected
- Test user workflows
- Validate data processing

### Usability Testing
- Evaluate user experience
- Assess interface intuitiveness
- Identify pain points

### Regression Testing
- Ensure existing functionality remains intact
- Test after code changes
- Validate bug fixes

### Performance Testing
- Measure response times
- Test under load
- Identify bottlenecks

## Defect Reporting

### Defect Classification
- **Critical**: Blocks core functionality
- **High**: Impairs major features
- **Medium**: Minor inconvenience
- **Low**: Cosmetic issues

### Report Structure
1. Summary
2. Steps to reproduce
3. Expected behavior
4. Actual behavior
5. Screenshots/videos
6. Environment details

## Checklist

- [ ] Test environment is set up
- [ ] Test cases are defined
- [ ] All features are tested
- [ ] Defects are documented with steps
- [ ] Screenshots are captured
- [ ] Reports are generated

## Output Deliverables

- Test summary report
- Defect tracking sheet
- Screenshots and replay videos
- Recommendations for improvement
