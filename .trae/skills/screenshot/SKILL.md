# Screenshot Capture

This skill provides system-level screenshot capabilities for visual documentation and debugging.

## Capture Options

### 1. Full Screen
- Capture entire display
- Multiple monitor support
- High-resolution output

### 2. Application Window
- Capture specific application
- Active window selection
- Window title bar included

### 3. Region Selection
- Custom rectangular region
- Precise dimensions
- Crosshair selection tool

## Output Formats

- PNG (lossless, transparent background)
- JPEG (compressed, smaller file size)
- GIF (animated sequences)
- BMP (raw, high quality)

## Usage Scenarios

### Documentation
- User guides and tutorials
- Feature documentation
- Technical manuals

### Debugging
- UI issue reporting
- Error message capture
- Layout verification

### Testing
- Visual regression testing
- UI comparison
- Test evidence

## Best Practices

1. **Capture Context**: Include relevant UI elements
2. **Annotations**: Add arrows and highlights
3. **Naming**: Use descriptive file names
4. **Quality**: Choose appropriate format
5. **Timing**: Capture at the right moment

## Checklist

- [ ] Capture method is appropriate
- [ ] Output format is correct
- [ ] Image quality is sufficient
- [ ] File is properly named
- [ ] Annotations are clear (if needed)
- [ ] Images are saved to correct location

## Output Deliverables

- Screenshot files
- Annotated images
- Visual documentation
